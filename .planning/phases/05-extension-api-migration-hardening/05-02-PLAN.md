<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension gate predicate → PhaseRunner | Gate predicate is caller-supplied code evaluated synchronously before handler(packet) and before FSM advancement |
| Extension lifecycle hook → PhaseRunner | Hook callbacks are caller-supplied code invoked around FSM advance |
| Extension provider check → provider-availability path | Provider check returns availability only; must not receive credentials or executors |
| Extension entry file → billing boundary walker | Extension modules are new entrypoints for the import-graph walk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | Gate predicate importing forbidden modules | mitigate | checkBillingBoundary() extended with extensionEntryFiles param; BILL-01 fired for any extension that reaches @anthropic-ai/claude-agent-sdk |
| T-05-02-02 | Tampering | Gate predicate mutating FSM state via Readonly ref | mitigate | evaluateGates() passes Readonly<FsmRunState>; TypeScript prevents writes through Readonly wrapper |
| T-05-02-03 | Tampering | Lifecycle hook mutating FSM state directly | mitigate | Hooks receive Readonly<FsmRunState> snapshot; veto result is a typed return, not a direct state mutation |
| T-05-02-04 | Spoofing | Provider check returning credentials or model session handles | mitigate | ProviderCheckRegistration.check() return type is ProviderAvailabilityResult (available/unavailable arrays only); TypeScript enforces at registration |
| T-05-02-05 | Information Disclosure | AdvisoryControlEvent validator accepts malformed events | mitigate | validateAdvisoryControlEvent() called before any control event leaves the SDK; rejects missing fields and unknown discriminants |
</threat_model>
