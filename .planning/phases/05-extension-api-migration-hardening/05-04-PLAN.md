<feature>
  <name>Append-only FSM rollback and schema migration functions</name>
  <files>sdk/src/advisory/fsm-rollback.ts, sdk/src/advisory/fsm-rollback.test.ts</files>
  <behavior>
    rollbackFsmState(input: { projectDir: string; workstream?: string }): Promise<FsmRollbackResult | RollbackBlockedEvent>
      - Constructs statePath = fsmStatePath(input.projectDir, input.workstream)
      - Asserts state initialized (stat directory; FsmStateError('init-required') if missing)
      - Acquires lock on statePath
      - Reads and parses state under lock
      - Finds last history entry with checkpoint === true (reverse scan)
      - If none found: releases lock; returns { kind: 'control', event: 'rollback-blocked', statePath, reason: 'no-checkpoint' }
      - Computes rolledBackEntryIds: all entry IDs after the checkpoint entry (entries that came after the checkpoint in chronological order)
      - Constructs rollback history entry: { entryId: randomUUID(), timestamp: now, fromState: state.currentState, toState: checkpointEntry.toState, runId: state.runId, outcome: 'rollback', configSnapshotHash: state.configSnapshotHash, rollbackEntry: { rollbackToEntryId: checkpointEntry.entryId, rolledBackEntryIds } }
      - Produces new immutable state: { ...state, currentState: checkpointEntry.toState, transitionHistory: [...state.transitionHistory, rollbackEntry], updatedAt: now }
      - Writes via writeFsmState(..., { heldLockPath: lockPath })
      - Returns FsmRollbackResult: { rollbackToEntryId: checkpointEntry.entryId, rolledBackEntryIds }

    migrateFsmState(input: { projectDir: string; workstream?: string }): Promise<FsmMigrateResult | AdvisoryControlEvent>
      - Constructs statePath
      - Asserts state initialized
      - Acquires lock
      - Reads raw file content; parses JSON (do NOT call parseFsmRunState — it would reject old versions)
      - Validates detectedVersion = parsed.stateSchemaVersion
      - If detectedVersion === CURRENT: return { kind: 'already-current' } (no-op, release lock)
      - If detectedVersion > CURRENT: return ResumeBlockedEvent (cannot migrate forward)
      - If detectedVersion < CURRENT (or missing → -1): perform migration:
          * Backfill entryId on all existing transitionHistory entries that lack it: uuid per entry
          * Set stateSchemaVersion: CURRENT_FSM_STATE_SCHEMA_VERSION (1)
          * Preserve all transitionHistory entries in order — never delete or reorder
          * Produce new immutable state object
          * Write via writeFsmState(..., { heldLockPath: lockPath })
          * Return FsmMigrateResult: { sourceVersion: detectedVersion, targetVersion: 1, entriesBackfilled: N }

    Test cases:
      - rollback with checkpoint → currentState restored; rollback entry appended; prior entries intact
      - rollback with no checkpoint → RollbackBlockedEvent returned; state file unchanged
      - rollback appends rollback entry with correct rollbackToEntryId and rolledBackEntryIds
      - migrate v0 state (stateSchemaVersion: 0) → history entries gain entryId; stateSchemaVersion becomes 1
      - migrate v1 state → already-current, no write
      - migrate state with stateSchemaVersion > 1 → ResumeBlockedEvent
      - rollback preserves all entries after checkpoint (never deletes)
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw FSM state file → migrateFsmState | Old-version state file content enters migration path; must be parsed without calling parseFsmRunState (which would throw) |
| Lock holder → writeFsmState | writeFsmState must always receive heldLockPath; calling without it acquires a second lock and deadlocks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Tampering | Rollback deleting history entries | mitigate | rollbackFsmState() uses append-only pattern: updatedState.transitionHistory = [...state.transitionHistory, rollbackEntry]; no entries are removed |
| T-05-04-02 | Denial of Service | Concurrent rollback race condition | mitigate | Lock acquired before read; writeFsmState called with { heldLockPath } preventing second lock acquisition; finally block always releases |
| T-05-04-03 | Tampering | migrateFsmState reordering history | mitigate | migrateFsmState preserves transitionHistory array order; only backfills missing entryId fields; no sorting or reordering |
| T-05-04-04 | Tampering | migrateFsmState calling parseFsmRunState (which rejects old versions) | mitigate | migrateFsmState reads raw JSON directly via readFile + JSON.parse before the version check; parseFsmRunState is NOT called on the old-version state |
| T-05-04-05 | Denial of Service | Init-required check missing before lock acquisition | mitigate | checkStateInitialized() runs before acquireFsmLock(); missing .planning/ directory returns FsmStateError('init-required') without attempting lock |
</threat_model>
