<feature>
  <name>Extension slot registry with register() and finalize()</name>
  <files>sdk/src/advisory/extension-registry.ts, sdk/src/advisory/extension-registry.test.ts</files>
  <behavior>
    Registration invariants (checked eagerly at register()):
    - ExtensionID pattern: must match ^[a-z][a-z0-9-]*$ for both extensionId part and localStepId part of namespaced IDs
    - Positional step references (integer anchors) are rejected immediately
    - Duplicate slot detected: same extensionId + same slot kind conflicts → ExtensionRegistryError('duplicate-id')
    - gsd.* and core.* extension ID prefixes → ExtensionRegistryError('reserved-prefix')
    - ReplaceInstructionRegistration type enforces instruction field only at TypeScript level (no agents, tools, evidence, transitions)

    Finalize validation (checked at finalize()):
    - Build before/after edge map across all InsertStepRegistration and GateRegistration slots
    - findCycle() on the edge map → hard CompileDiagnostic error with full chain if cycle found
    - Unknown dependency ID (not in registered set) → hard CompileDiagnostic error
    - Co-anchored extensions with same anchor and no declared relative ordering → CompileDiagnostic WARNING 'unordered-co-anchor'; registration order is the tie-break
    - Returns SealedExtensionGraph (sealed — no further registration after finalize())

    SealedExtensionGraph surfaces:
    - insertedStepsFor(anchorStepId: string, position: 'before' | 'after'): InsertStepRegistration[]
    - instructionReplacementFor(stepId: string): ReplaceInstructionRegistration | undefined
    - evaluateGates(stepId: string, snapshot: Readonly<FsmRunState>): { kind: 'pass' } | { kind: 'gate-failed'; extensionId: string }
    - lifecycleHooks(): LifecycleHookRegistration[]
    - providerChecks(): ProviderCheckRegistration[]

    Cases:
    - register(validInsert) → no throw, slot stored
    - register(duplicateId) → throws ExtensionRegistryError('duplicate-id')
    - register(positionalRef) → throws ExtensionRegistryError('invalid-slot-shape')
    - register(reservedPrefix: 'gsd/step') → throws ExtensionRegistryError('reserved-prefix')
    - register(invalidNamespace: 'MyExt/step') → throws ExtensionRegistryError('invalid-namespace')
    - finalize() with cycle A→B→A → throws or returns diagnostics with cycle chain
    - finalize() with unknown dep → diagnostics with unknown dep message
    - finalize() with co-anchor no order → diagnostics WARNING unordered-co-anchor
    - finalize() clean → returns SealedExtensionGraph
    - evaluateGates() gate predicate returns true → { kind: 'pass' }
    - evaluateGates() gate predicate returns false → { kind: 'gate-failed', extensionId }
  </behavior>
  <implementation>
    1. Write RED tests first. Run suite — all must FAIL.
    2. Implement ExtensionRegistryError, slot types, ExtensionRegistry class, SealedExtensionGraph.
    3. Copy findCycle from validators.ts (it is not exported — copy inline).
    4. Run suite — all must PASS.
    5. Ensure TypeScript types structurally prohibit non-instruction fields on ReplaceInstructionRegistration.

    Design notes:
    - GateRegistration.predicate signature: (snapshot: Readonly<FsmRunState>, meta: { stepId: string; extensionId: string }) => boolean
    - finalize() returns SealedExtensionGraph (does not mutate the registry — allow re-call for testing)
    - SealedExtensionGraph stores sorted ordered arrays for deterministic evaluation
    - WARNING diagnostics for unordered-co-anchor are returned as CompileDiagnostic[] on SealedExtensionGraph.warnings, not thrown
    - No external imports — pure TypeScript with no fs/promises, no shell calls
    - EXTENSION_ID_PATTERN = /^[a-z][a-z0-9-]*$/
    - Namespaced IDs: split on first '/' — validate both sides against EXTENSION_ID_PATTERN
    - Reserved prefixes: ['gsd', 'core']
  </implementation>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension author → SDK | Extension slot registrations arrive as untrusted caller-supplied values |
| Gate predicate → FSM state | Gate predicate receives a Readonly snapshot; must not receive a mutable reference |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | ExtensionRegistry.register() | mitigate | Validate extensionId namespace, slot shape, and reserved prefixes at register() call time; throw ExtensionRegistryError with specific code |
| T-05-01-02 | Tampering | ReplaceInstructionRegistration | mitigate | TypeScript structural type constrains the slot to instruction field only; compile error if extension author includes agents/tools/evidence |
| T-05-01-03 | Tampering | Gate predicate receiving mutable FSM state | mitigate | evaluateGates() passes Readonly<FsmRunState>; TypeScript prevents write access through the readonly wrapper |
| T-05-01-04 | Denial of Service | Extension dependency cycle | mitigate | findCycle() at finalize() detects cycles before any packet emission; hard diagnostic with full chain |
| T-05-01-05 | Elevation of Privilege | Reserved prefix squatting (gsd.*, core.*) | mitigate | register() rejects extensionIds starting with 'gsd' or 'core' prefix with ExtensionRegistryError('reserved-prefix') |
</threat_model>
