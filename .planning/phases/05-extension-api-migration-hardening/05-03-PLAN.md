<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw FSM state file content → parseFsmRunStateOrControlEvent | Untrusted file content enters the version-check path |
| advanceFsmState caller → entryId generation | Caller must not supply its own entryId; generation must be SDK-internal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | Caller supplying its own entryId | mitigate | entryId is generated inside advanceFsmState() with randomUUID(); FsmTransitionInput does not accept entryId |
| T-05-03-02 | Tampering | parseFsmRunStateOrControlEvent returning partial state for version mismatch | mitigate | Version check runs before parseFsmRunState() is called; mismatched version always returns a typed control event, never a partial FsmRunState |
| T-05-03-03 | Information Disclosure | Migration steps containing executable instructions | mitigate | migrationSteps type is Array<{ description: string }> — no callable fields; TypeScript enforces at construction |
| T-05-03-04 | Denial of Service | Malformed JSON crashing on parse | mitigate | parseFsmRunStateOrControlEvent wraps JSON.parse in try/catch; throws FsmStateError('read-failed') rather than unhandled exception |
</threat_model>
