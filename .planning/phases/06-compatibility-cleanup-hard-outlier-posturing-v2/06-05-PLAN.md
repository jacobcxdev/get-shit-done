## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Eligibility verdict → pilot decision | A pass verdict from --check-slim-eligibility is required before any workflow file is touched; verdict cannot be overridden by the executor |
| Human checkpoint → pilot scope | The user must explicitly list approved workflowIds; the executor must not infer or expand the approved set |
| Thin launcher content → compiler | The launcher file contains only the gsd-advisory YAML block; it must not contain workflow prose or hidden instructions |
| docs/archive/ → compiler | Archived prose files are read-only historical reference; the compiler must not treat them as executable goldens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05-01 | Elevation of Privilege | slimming a hard-outlier workflow | mitigate | --check-slim-eligibility rejects hard-outlier workflowIds with OUTL-01 diagnostic; pilot pre-flight check enforces pass verdict before any write |
| T-06-05-02 | Tampering | archived prose used as executable golden | mitigate | docs/archive/ files are write-once historical references; compiler infers semantics from launcher metadata and generated manifests, not archived prose — enforced by RESEARCH.md planning boundary |
| T-06-05-03 | Tampering | launcher file containing hidden workflow prose | mitigate | validateLauncherMetadata (Plan 03) and extractLauncherBlock reject any file with text outside the fenced block; parity gate re-runs eligibility on launchers |
| T-06-05-04 | Repudiation | pilot BLOCKED without reason | mitigate | BLOCKED path explicitly records which gates failed for which candidates in the plan summary |
| T-06-05-05 | Denial of Service | parity gate fails after launcher write | mitigate | If node scripts/phase4-parity.cjs exits non-zero after the archive move, the pilot files are reverted to original prose and the failure reason is recorded |
