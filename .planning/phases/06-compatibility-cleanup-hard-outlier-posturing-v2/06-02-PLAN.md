    ```typescript
    function evaluateGate(
      gate: SlimEligibilityGate,
      evidence: string[],
      diagnostics: CompileDiagnostic[],
    ): SlimEligibilityGateResult {
      if (diagnostics.some(d => d.severity === 'error')) {
        return { gate, status: 'fail', evidence, diagnostics };
      }
      if (evidence.length === 0) {
        return { gate, status: 'indeterminate', evidence, diagnostics };
      }
      return { gate, status: 'pass', evidence, diagnostics };
    }
    ```

    Main export:
    ```typescript
    export function evaluateSlimEligibility(
      workflowId: string,
      report: CompileReport,
      parityIndexPath?: string,   // optional override for testing; default = absolute generated path
    ): SlimEligibilityVerdict
    ```

    Implementation sequence:

    1. Hard-outlier short-circuit: find ClassificationEntry whose workflowId === workflowId. If entry.isHardOutlier:
       - Emit OUTL-01 diagnostic: mkError('OUTL-01', 'outlier', entry.commandId, entry.outlierPostureRecord?.posturePath ?? 'sdk/src/advisory/outlier-postures/', `hard-outlier workflow ${workflowId} cannot be slimmed; see posture record`)
       - Return { workflowId, commandId: entry.commandId, eligible:false, status:'fail', isHardOutlier:true, posturePath: entry.outlierPostureRecord?.posturePath, gates:[], diagnostics: diag }

    2. Unknown workflow short-circuit: find WorkflowEntry with id === workflowId in report.manifests.workflows. If absent:
       - Emit SLIM-01 diagnostic
       - Return { workflowId, eligible:false, status:'fail', isHardOutlier:false, gates:[], diagnostics: diag }

    3. typed-transitions gate: check if report.manifests.workflowSemantics has an entry for workflowId, validate its semantic families non-empty. Current policy: all Markdown-inferred semantics are treated as INDETERMINATE because they are prose-derived, not durable typed transitions. Evidence = [] → status:'indeterminate'. Add a comment: "// indeterminate until durable non-prose transition evidence surface exists (RESEARCH.md §typed-transitions)". This is intentional fail-closed behaviour per D-02.

    4. packet-sequencing gate: evidence = [] always for now, because live compile has no packet sequence inventory. Status: 'indeterminate'. Add comment: "// indeterminate until packet definitions are collected by compiler (RESEARCH.md §packet-sequencing)".

    5. provider-routing gate:
       - Find classEntry for this workflowId (may be null if no command maps to this workflow).
       - If classEntry exists and classEntry.agentTypes is empty: evidence = ['no-agent-types-declared: no provider-specific route required'], status:'pass'.
       - If classEntry exists and agentTypes non-empty: check if report.manifests.workflowSemantics has mandatoryProviders for this workflow. If mandatoryProviders present and non-empty: evidence = mandatoryProviders.map(p => `mandatory-provider:${p}`), status:'pass'. If absent or empty agentTypes with no route: status:'indeterminate'.
       - If no classEntry: status:'indeterminate'.
       Emit SLIM-01 warning (not error) if indeterminate: mkWarning('SLIM-01', 'slim', workflowId, workflowId, 'provider-routing evidence indeterminate').

       - Otherwise: evidence = [`parity-tier:${entry.tier}`], status:'pass'.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → parseCompileArgs | --check-slim-eligibility value is an untrusted string from the user; must be treated as a workflowId to look up, not executed |
| slim-eligibility.ts → generated parity JSON | parity-workflow-index.json is a generated file; must be read defensively (parse errors → fail-closed) |
| evaluateSlimEligibility → CompileReport | CompileReport is compiler-owned; no external input crosses this boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | verdict.eligible flag | mitigate | eligible is derived from gates array — never set independently; any gate fail/indeterminate forces eligible:false |
| T-06-02-02 | Elevation of Privilege | hard-outlier bypass via workflowId lookup | mitigate | hard-outlier check uses ClassificationEntry.isHardOutlier from compiler manifest, not caller-supplied metadata; cannot be bypassed by passing a non-canonical ID |
| T-06-02-03 | Information Disclosure | parity-workflow-index.json read error in verdict JSON | accept | Generated file path is a local development artifact; no sensitive data in verdict JSON beyond workflow IDs |
| T-06-02-04 | Spoofing | fake parity index via parityIndexPath override | accept | parityIndexPath override is for testing only; production path is hardcoded in the function default; no external caller can change the default without modifying source |
| T-06-02-05 | Denial of Service | unreadable parity-workflow-index.json | mitigate | File read errors produce SLIM-01 fail verdict and non-zero exit rather than throwing unhandled exception |
