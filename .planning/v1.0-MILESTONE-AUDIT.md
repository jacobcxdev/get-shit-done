### Unsatisfied Requirement

#### STAT-04 — Flat `_auto_chain_active` still affects live CJS plan-phase init

**Requirement:** `workflow._auto_chain_active` no longer exists as a flat shared key; its semantics are captured per-workstream inside FSM run-state.

**Evidence:**

- `get-shit-done/bin/lib/init.cjs:211` constructs `legacyAutoChainKey = ['_auto', 'chain', 'active'].join('_')`.
- `get-shit-done/bin/lib/init.cjs:267` returns `auto_chain_active: !!(rawConfig.workflow?.[legacyAutoChainKey])`.
- The SDK query path had migrated to FSM-scoped `autoMode`, so this creates a split-brain path: SDK reads FSM state, while CJS plan-phase init can still read stale config.

**Impact:** plan-phase auto-advance routing can be driven by `.planning/config.json` rather than `.planning/fsm-state.json` or workstream-scoped FSM state, violating the durable state contract and risking unintended auto-chain behaviour.

**Required fix:** remove legacy flat config semantics from `cmdInitPlanPhase()`, derive `auto_chain_active` from the same FSM-scoped source as `check.auto-mode`, and update/remove tests that still encode config-derived `_auto_chain_active` behaviour.
